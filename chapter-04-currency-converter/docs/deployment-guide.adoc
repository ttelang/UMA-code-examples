This document covers WebAssembly (WASM) compilation targets, WASI runtime versions, and how to run Universal Microservices in browser environments.

== WebAssembly Compilation Targets

=== wasm32-wasip1 (Recommended)

The `wasm32-wasip1` target compiles Rust code to WebAssembly with **WASI Preview 1** support.

[cols="1,3"]
|===
|Target |`wasm32-wasip1`
|WASI Version |Preview 1 (snapshot_preview1)
|Status |Stable, production-ready
|Rust Support |Stable toolchain
|Runtime Support |Universal (wasmtime, wasmer, browser polyfills, edge workers)
|Rust Command |`rustup target add wasm32-wasip1`
|===

==== Capabilities

* File I/O (stdin, stdout, stderr)
* Environment variables
* Command-line arguments
* System time and random number generation
* Basic networking (limited)

==== Example Build

[source,bash]
----
# Add the target
rustup target add wasm32-wasip1

# Compile your binary
cargo build --bin fx --target wasm32-wasip1 --release

# Output location
# target/wasm32-wasip1/release/fx.wasm
----

=== wasm32-wasi (Deprecated Alias)

`wasm32-wasi` is the **original name** for the same target now called `wasm32-wasip1`. Both names work identically:

[source,bash]
----
# These produce identical output:
cargo build --target wasm32-wasi
cargo build --target wasm32-wasip1
----

IMPORTANT: Use `wasm32-wasip1` going forward for explicit version clarity.

=== wasm32-wasip2 (Experimental)

The `wasm32-wasip2` target compiles to **WASI Preview 2** (Component Model).

[cols="1,3"]
|===
|Target |`wasm32-wasip2`
|WASI Version |Preview 2 (Component Model)
|Status |Experimental (2023+)
|Rust Support |Nightly builds required
|Runtime Support |Limited (wasmtime 14+ partial support)
|Rust Command |Not available in stable yet
|===

==== Component Model Benefits

* **Better composability** - Align with UMA orchestration patterns
* **Interface types** - Cleaner contract boundaries
* **Virtualization** - Enhanced trust boundary isolation
* **Higher-level APIs** - `wasi:http`, `wasi:cli`, `wasi:filesystem`

TIP: Preview 2's Component Model is architecturally aligned with UMA's long-term vision, but Preview 1 is production-ready today.

== WASI Versions Comparison

=== WASI Preview 1 (Current Standard)

[cols="1,3"]
|===
|Released |2019-2023
|Architecture |Module-based (traditional WASM imports/exports)
|APIs |Low-level syscalls: `fd_read`, `fd_write`, `clock_time_get`, `random_get`
|Adoption |Widely deployed in production systems
|Runtimes |wasmtime, wasmer, browser polyfills, Cloudflare Workers, Fastly Compute
|===

==== System Calls (Partial List)

[source,rust]
----
// stdin/stdout I/O
fd_read(fd: i32, iovs: *const iovec, iovs_len: usize, nread: *mut usize) -> errno
fd_write(fd: i32, iovs: *const ciovec, iovs_len: usize, nwritten: *mut usize) -> errno

// Time and randomness
clock_time_get(id: clockid, precision: timestamp, time: *mut timestamp) -> errno
random_get(buf: *mut u8, buf_len: usize) -> errno

// Process control
proc_exit(rval: exitcode) -> !
----

[cols="1,3"]
|===
|Released |2023+ (in development)
|Architecture |Component-based with interface types
|APIs |High-level interfaces: `wasi:http/outgoing-handler`, `wasi:cli/terminal-output`
|Adoption |Experimental, early adopters only
|Runtimes |wasmtime 14+ (partial), experimental implementations

== Which WASI Version for UMA?

=== Current Recommendation: WASI Preview 1

For Universal Microservices in 2026, **use wasm32-wasip1**:

✅ **Universal runtime support** - Runs identically across:

* Native execution (development/testing)
* Cloud runtimes (wasmtime, wasmer)
* Edge computing platforms (Cloudflare Workers, Fastly Compute)
* Browsers (with JavaScript WASI polyfill)

✅ **Stable tooling** - Mature Rust compiler, no nightly builds required

✅ **Proven portability** - This repository demonstrates behavioral parity between native and WASM execution (see `chapter-06-portability-lab`)

✅ **Production-ready** - Used in real-world edge computing today

=== Future Migration: WASI Preview 2

Preview 2's **Component Model** better aligns with UMA's architectural principles:

* **Composability** - Natural fit for UMA orchestration
* **Interface contracts** - Explicit boundaries match UMA's contract-driven design
* **Virtualization** - Enhanced isolation for trust boundaries
* **Evolution** - Better support for versioned interfaces

However, Preview 2 is still experimental. The UMA *pattern* remains valid—you'd simply swap compilation targets when Preview 2 matures.

=== Data Flow

. **User Input** → HTML form collects currency conversion parameters
. **JavaScript Adapter** → Serializes input to JSON, encodes as UTF-8 bytes
. **WASI Polyfill** → Provides stdin bytes to WASM module
. **WASM Execution** → Rust binary reads stdin, processes request, writes stdout
. **JavaScript Capture** → Collects stdout bytes from polyfill
. **Result Display** → Decodes UTF-8, parses JSON, updates UI

[source,javascript]
----
// Simplified flow
const input = { from: "USD", to: "EUR", amount: 100 };
const inputBytes = new TextEncoder().encode(JSON.stringify(input));

const stdoutBytes = [];
const instance = await WebAssembly.instantiate(wasmBytes, {
  wasi_snapshot_preview1: createPolyfill(inputBytes, stdoutBytes)
});

instance.exports._start(); // Execute Rust main()

const output = JSON.parse(new TextDecoder().decode(stdoutBytes));
// { from: "USD", to: "EUR", amount: 100, rate: 0.92, result: 92.0 }
----

==== src/index.js

[source,javascript]
----
// Currency converter Worker for Cloudflare
export default {
  async fetch(request, env) {
    // Only accept POST requests
    if (request.method !== 'POST') {
      return new Response('Method not allowed', { status: 405 });
    }

    try {
      // Parse the incoming JSON request
      const input = await request.json();
      
      // Validate input structure
      if (!input.from || !input.to || typeof input.amount !== 'number') {
        return new Response(JSON.stringify({
          error: 'Invalid input: requires {from, to, amount}'
        }), {
          status: 400,
          headers: { 'Content-Type': 'application/json' }
        });
      }

      // Create WASI imports for the module
      const stdinBytes = Array.from(
        new TextEncoder().encode(JSON.stringify(input))
      );
      const stdoutBytes = [];
      let memory;

      const wasiImports = createWasiImports(
        { get buffer() { return memory.buffer; } },
        stdinBytes,
        stdoutBytes
      );

      // Instantiate the WASM module (env.FX_WASM is bound via wrangler.toml)
      const instance = await WebAssembly.instantiate(
        env.FX_WASM,
        wasiImports
      );
      
      memory = instance.exports.memory;

      // Execute the conversion
      try {
        instance.exports._start();
      } catch (err) {
        if (err.name === 'WasmExit') {
          // Handle exit codes
          if (err.code !== 0) {
            const errorMap = {
              1: 'Invalid input JSON',
              2: 'Unsupported currency pair',
              3: 'Output write error',
              4: 'Input read error'
            };
            return new Response(JSON.stringify({
              error: errorMap[err.code] || `Process exited with code ${err.code}`
            }), {
              status: 400,
              headers: { 'Content-Type': 'application/json' }
            });
          }
        } else {
          throw err;
        }
      }

      // Parse and return the result
      const outputStr = new TextDecoder().decode(new Uint8Array(stdoutBytes));
      const result = JSON.parse(outputStr);

      return new Response(JSON.stringify(result), {
        status: 200,
        headers: {
          'Content-Type': 'application/json',
          'Cache-Control': 'public, max-age=3600' // Cache for 1 hour
        }
      });

    } catch (error) {
      return new Response(JSON.stringify({
        error: error.message
      }), {
        status: 500,
        headers: { 'Content-Type': 'application/json' }
      });
    }
  }
};

// WASI polyfill implementation (same as browser adapter)
function createWasiImports(memory, stdinBytes, stdoutBytes) {
  return {
    wasi_snapshot_preview1: {
      fd_write: (fd, iovsPtr, iovsLen, nwrittenPtr) => {
        if (fd !== 1 && fd !== 2) return 52;
        const memU32 = new Uint32Array(memory.buffer);
        const memU8 = new Uint8Array(memory.buffer);
        let bytesWritten = 0;
        const start = iovsPtr >>> 2;
        for (let i = 0; i < iovsLen; i++) {
          const offset = memU32[start + i * 2];
          const length = memU32[start + i * 2 + 1];
          stdoutBytes.push(...memU8.slice(offset, offset + length));
          bytesWritten += length;
        }
        memU32[nwrittenPtr >>> 2] = bytesWritten;
        return 0;
      },
      fd_read: (fd, iovsPtr, iovsLen, nreadPtr) => {
        if (fd !== 0) return 52;
        const memU32 = new Uint32Array(memory.buffer);
        const memU8 = new Uint8Array(memory.buffer);
        let bytesRead = 0;
        const start = iovsPtr >>> 2;
        for (let i = 0; i < iovsLen; i++) {
          const offset = memU32[start + i * 2];
          const length = memU32[start + i * 2 + 1];
          const chunk = stdinBytes.slice(0, length);
          memU8.set(chunk, offset);
          stdinBytes.splice(0, chunk.length);
          bytesRead += chunk.length;
          if (stdinBytes.length === 0) break;
        }
        memU32[nreadPtr >>> 2] = bytesRead;
        return 0;
      },
      fd_close: () => 0,
      fd_fdstat_get: () => 0,
      fd_prestat_get: () => 8,
      fd_prestat_dir_name: () => 8,
      args_sizes_get: (argcPtr, argvBufSizePtr) => {
        const memU32 = new Uint32Array(memory.buffer);
        memU32[argcPtr >>> 2] = 0;
        memU32[argvBufSizePtr >>> 2] = 0;
        return 0;
      },
      args_get: () => 0,
      environ_sizes_get: (envCountPtr, envBufSizePtr) => {
        const memU32 = new Uint32Array(memory.buffer);
        memU32[envCountPtr >>> 2] = 0;
        memU32[envBufSizePtr >>> 2] = 0;
        return 0;
      },
      environ_get: () => 0,
      random_get: (bufPtr, bufLen) => {
        const memU8 = new Uint8Array(memory.buffer);
        crypto.getRandomValues(memU8.subarray(bufPtr, bufPtr + bufLen));
        return 0;
      },
      clock_time_get: (_id, _precision, timePtr) => {
        const nowNs = BigInt(Date.now()) * 1000000n;
        const memView = new DataView(memory.buffer);
        memView.setBigUint64(timePtr, nowNs, true);
        return 0;
      },
      proc_exit: (code) => {
        const err = new Error('Process exited');
        err.name = 'WasmExit';
        err.code = code;
        throw err;
      }
    }
  };
}

[source,javascript]
----
// Lambda@Edge handler
exports.handler = async (event, context) => {
    const request = event.Records[0].cf.request;
    
    if (request.method !== 'POST') {
        return {
            status: '405',
            statusDescription: 'Method Not Allowed',
            body: 'Only POST requests accepted'
        };
    }
    
    // Decode base64 body (API Gateway integration)
    const body = Buffer.from(request.body.data, 'base64').toString('utf8');
    const input = JSON.parse(body);
    
    // Load WASM module (bundled with deployment package)

        status: '200',
        statusDescription: 'OK',

        },
        body: JSON.stringify(result)
